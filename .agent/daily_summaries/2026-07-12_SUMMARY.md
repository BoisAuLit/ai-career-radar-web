### Core design

- **AgentOps-3b is design/spec-only.** No
  planner code, no runner, no daemon, no
  Playwright harness, no benchmark
  fixtures.
- **Codex becomes a manual-first,
  read-only daily planner.** Invoked
  via Codex CLI (ChatGPT sign-in per
  AgentOps-2b Q7 — **not** OpenAI API
  in any Q7-blocked sense).
- **Codex should reduce Bohao's daily
  planning burden but not replace
  Human + ChatGPT judgment.** Every
  planner report is a draft
  recommendation the human is free to
  override, redirect, or discard.
- **Codex may eventually write

  `.agent/planner_reports/`**, but

### Planner report schema (14 subsections)

Every future planner report contains
exactly these subsections in order:

1. Date / repo status
2. Yesterday / recent completed work
3. Current blockers
4. Current automation phase
5. Product state
6. Candidate next tasks (2-4)
7. Recommended single next task
8. Why this task / why not
   alternatives
9. Risk classification
10. Required validation
11. Whether report regression is
    required
12. Human decisions needed
13. Exact Claude Code prompt if
    approved
14. Stop conditions / forbidden
    actions

**Target: ≤ 100 lines / ≤ 15 minutes of
human + ChatGPT review time.**

### Command boundaries

**Allowed** (all read-only):

- `git status`, `git log --oneline
  -N`, `git diff --name-only
  origin/main..HEAD`, `git rev-list
  --left-right --count
  origin/main...main`, `git show
  --stat <SHA>`
- Read-only `.agent/` inspection:
  `find` / `grep` / `ripgrep` /
  `cat` / `head` / `tail` / `wc` /
  `sed -n 'A,Bp' file`
- `python3 -c "import json; …"` for

**Forbidden**:

- `git add` / `commit` / `push` /
  `reset` / `checkout` / `merge` /
  `rebase` / `revert` / `stash` /
  `tag` / `branch -D`
- `npm install` / `ci` / `uninstall`
  / `update` / `yarn` / `pnpm`
  anything
- `npm run dev` / `screenshot` /
  `build` unless explicitly
  requested
- Collector (`npm run collect` /
  `python -m scripts.collector …`)
- Deploy (`vercel deploy` /
  anything)
- `.env*` access
- OpenAI / Anthropic API HTTP calls
- Playwright report generation in
  3b (that's 3e's scope)

### Future regression integration (5 states)

- **A. Harness unavailable
  (current)** → mark
  "unavailable — this task's
  product-quality impact is not
  auto-verifiable". Do NOT
  pretend coverage exists.
- **B. Not required** → task
  class doesn't touch generated-
  report quality (safe for docs /
  `.agent`-only cases per
  AgentOps-3a §8.2).
- **C. Green last verdict +
  required** → mark "required;
  last verdict green (SHA
  <baseline>); run before push".
- **D. Amber last verdict** →
  escalate to Human + ChatGPT.
  Do NOT auto-recommend
  proceeding.
- **E. Red last verdict** → MUST
  recommend fix / revert, never
  new feature work.
