## Open risks

1. **Hero mock attribution numbers are hardcoded.** The strings
   `92 applied_ai JDs` and `5 evidence quotes` in the homepage hero mock
   card are marketing-surface copy that does not track the live corpus.
   Status: accepted for now; revisit only if/when the mock is expected to
   read like live data.
2. **Report action bar lives inside the report shell.** Commit `77a8e70`
   nested the Copy/Download/Eval/Start Over band inside the report
   `<section>`. If future work needs to render the action bar without the
   shell (e.g., a sticky-on-scroll variant), the markup will need to be
   split into a reusable component. Status: open; no immediate plan.
3. **Displayed model string is static.** The footnote under Generate now
   reads `Sonnet 4.6 + Haiku 4.5` but is not linked to
   `src/lib/anthropic.ts`. If the model selection changes in code, this
   caption will silently drift. Status: open; small refactor candidate.
4. **`new_task.py` covers TASK files only.** A similar helper for
   RUN_REPORT or DECISION files might also help, but is **deferred**
   until manual friction proves it's worth the extra surface area.
5. **Keep `new_task.py` local-only.** Do not let it grow into an
   orchestrator (no subprocess, no network, no LLM calls). If we ever
   want orchestration, it goes in its own AgentOps-N phase with its
   own design memo. Status: enforced via TASK stop conditions and
   policy §4.
6. **Do not add OpenAI API automation yet.** Several more manual
   AgentOps runs need to be stable first so we accumulate good
   few-shot DECISIONs. Status: deferred per `.agent/README.md`'s
   AgentOps-0 non-goals.
