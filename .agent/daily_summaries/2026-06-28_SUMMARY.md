1. **Keep the helper scripts as file-template helpers only.** Their
   only job is to scaffold a single file from a single template per
   invocation. Anything that orchestrates *between* files (auto-running
   the next step, watching directories, etc.) is out of scope.

   without a separate design memo.** If a future task proposes shared
   state, scheduling, or "run the whole loop" semantics, it requires
   its own design memo and `risk_level: red` approval. The current
   helpers are deliberately three independent scripts that each do one
   small thing.

   walks `tasks/`, `run_reports/`, `decisions/` and flags any
   `task_id` missing one of the three would be useful as a quick
   audit. Only add it *if* a missing file actually happens; do not
   pre-build it.

   the marginal value is tiny (the duplication is ~15 lines of date
   validation). Skip until a fourth helper appears.
