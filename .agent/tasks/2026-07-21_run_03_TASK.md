## Matching tiers (5c refinements applied)

For each evidence quote with cited source id:

1. Verbatim substring in source body → **GREEN** `verbatim`
2. Whitespace / punctuation / dash / ellipsis-normalized substring →
   **AMBER** `normalized`
3. Case-insensitive normalized substring → **AMBER** `case_insensitive`
4. If quote contains literal `...` OR Unicode `…`:
   - split on ellipsis · trim · drop trivial fragments (<12
     non-space chars OR <3 words)
   - require **every** meaningful fragment to match same source body
     through tier 1/2/3
   - all fragments match → **AMBER** `ellipsis_fragments`
   - any meaningful fragment unmatched → **RED** `unmatched_ellipsis_fragment`
5. Else → **RED** `unmatched`
6. Unresolved source id → **RED** `unresolved_source_id`

For quote-like evidence without cited source id:

- Unique verbatim → **AMBER** `missing_source_id_unique_verbatim`
- Unique normalized/case-insensitive → **AMBER**
  `missing_source_id_unique_normalized`
- Multiple matches → **AMBER** `ambiguous_source_id`
- No match → **RED** `unmatched_missing_source_id`

Metadata checks: company case+punct-insensitive containment · role
loose containment · appendix cross-check · duplicates AMBER-only.
