## Objective

Tune `scripts/report-regression-local.mjs` so it captures the **actual generated
report content** (the `<ReactMarkdown>` output region) instead of the whole page
`body.innerText`, then re-run **Fixture A** once locally against `localhost:3000`
and write a new small verdict. Primary goal is **correct capture scope**, not
forced green. Length band stays 1500-6000 initially; if selected report content
still exceeds that after correct capture, the verdict is allowed to remain AMBER
as a length-band calibration finding — that is not a capture bug.

## Background

AgentOps-3e (`2026-07-12_run_03`) shipped the first minimal local Playwright
regression prototype. The prototype's first real run produced verdict = **AMBER**
purely because `report_length_in_soft_band` failed with `chars=17181`. Root cause:
the harness captures `page.locator("body").innerText()`, which includes the hero,
form, report card, methodology strip, attribution, and footer — not only the
generated report. All other red checks (page loaded / resume filled / target
filled / generate clicked / done reached / incomplete banner absent / all 5
required sections present / Evidence Appendix present / must-not-happen zero /
duration under hard cap / no fatal error / no production target) **passed**. The
prior DECISION (`.agent/decisions/2026-07-12_run_03_DECISION.md`) explicitly
endorsed 3e-tune as the default next loop and preferred fix = narrow capture via
container locator, alternative = widen length band. This loop implements the
preferred fix.
