## Objective

Add a lightweight **validation script** that verifies
`src/lib/web-bundle-stats.ts` matches
`src/data/web_bundle.json` and fails clearly if the
manually synced helper drifts. This closes the
"manually synced" risk called out in the P2.0b
DECISION §risks (`WEB_BUNDLE_STATS` has 6 constants
and no build-time guardrail against silent
divergence). No codegen yet, no build gate yet —
just an explicit drift check, invokable via
`node scripts/check-web-bundle-stats.mjs` or
`npm run check:web-bundle-stats`.
