## Objective

Create a formal automation policy + reporting framework so that the
**future** A/B time-window operating model can run safely:

- **A · Automation Time** — weekdays 09:00–19:00 working hours, daily
  01:00–09:00 sleep window, all-day weekends. Codex CLI plans /
  decides / reviews; Claude Code executes coding tasks.
- **B · Non-Automation Time** — short human review window with
  ChatGPT Chat: read the automation report, approve blocked items if
  appropriate, set next direction.

The deliverables of the future implementation step (G-1 below) are
five files under `.agent/`. None of them runs anything; they only
describe how a runner WOULD operate and how a human + ChatGPT WOULD
review the result.

**This TASK does NOT build the runner.** A separate AgentOps-2b TASK
will scope a small, safe MVP runner once this policy is approved.

## Background

Where we are after today:
- AgentOps file protocol exists (`.agent/README.md`, policy, templates,
  helpers `new_task.py` / `new_run_report.py` / `new_decision.py`,
  three full loops run end-to-end including a cross-repo G2.1b/c).
- The human has run every loop manually — drafting TASKs, copying
  outputs to ChatGPT, approving merges. Sustainable for a few loops
  per day, not sustainable as a daily operating rhythm.

What's changing:
- Add Codex CLI as a **planner / reviewer** during Automation Time
  (GPT-5.5, reasoning effort **high**, NOT extra-high).
- Keep Claude Code as the **executor** during Automation Time.
- Keep ChatGPT Chat as the **external reviewer / strategist** during
  Non-Automation Time.
- Keep the human as the **final owner and red-zone approver**.
- Do NOT introduce OpenAI API. Do NOT expand GitHub Actions. Do NOT
  add cron. Do NOT add an autonomous daemon. Do NOT auto-approve red.

## Constraints

- **Documentation / policy only.** No runner, no daemon, no scheduler,
  no watcher, no GitHub Action.
- **No OpenAI API integration.** No SDK install, no API key, no `.env`
  entry, no HTTP request to OpenAI. (Defer to a separate AgentOps-N
  with its own design memo if ever needed.)
- **No Codex / Claude Code config mutation.** The policy specifies
  expected behavior; it does not push settings to either tool.
- **No expansion of `.github/workflows/`.** Daily cron continues to
  fire on its existing schedule; no new workflows added.
- **No new cron, no new deploy hook.**
- **No npm changes / no new deps.**
- **No pipeline repo changes** in either AgentOps-2a TASK creation OR
  the future execution step.
- **No autonomous "run forever" loop.** Each automation window has a
  defined end; the runner stops at end-of-window even if work remains.
- **No red-zone auto-approval.** The policy explicitly carves out
  red-zone work as human-only territory; no Codex / Claude Code edit
  proceeds without human GO for those paths.
- **Standing daily review.** Non-Automation Time must include a human
  + ChatGPT Chat read-through of the automation report; the policy
  requires the report to exist before the next Automation Window opens.
- **No push, no deploy** during this TASK's execution. Web-repo TASK +
  RUN_REPORT commit straight to `main` and stay unpushed until human
  approval.

## Implementation outline (for the executor)

The executor (AgentOps-2a-run) writes 5 files. Each described below;
total budget ~600-900 lines of new markdown across the five files.

### File 1 · `.agent/policies/automation_policy.md` (~250-350 lines)

The load-bearing operating contract. Sections (mirror this TASK's spec):

1. **Roles** — Human owner / ChatGPT Chat / Codex CLI / Claude Code
   / GitHub / origin / GitHub Actions / OpenAI API (deferred).
   For each: responsibility, what they CAN do, what they MUST NOT do,
   what they DEFER to.
2. **Automation windows** —
   - Workday automation: weekdays 09:00–19:00 (human is working but
     not actively driving the agent loop).
   - Sleep automation: every day 01:00–09:00.
   - Weekend automation: Saturday + Sunday all day.
   - Non-automation review window: the human's "review with ChatGPT
     Chat" slot, target ≤ 30 minutes/day.
3. **Codex CLI rules** —
   - Model: GPT-5.5.
   - Reasoning effort: `high` for planning/review/decision. **NOT**
     `extra-high` (cost/latency disincentive).
   - Acts as planner / reviewer / decision proposer.
   - Does NOT make final red-zone approvals.
   - Stops + writes to `blockers.md` when risk exceeds policy.
   - Does NOT introduce OpenAI API.
4. **Claude Code rules** —
   - Executes TASK files.
   - Writes RUN_REPORT; may draft DECISION (human/ChatGPT confirms).
   - Runs validations declared in the TASK.
   - Does NOT push / deploy / merge red-zone without explicit approval.
   - Does NOT touch forbidden files.
   - Does NOT run classifier / extractor / collection unless the TASK
     explicitly allows.
   - Quota guidance: spend on valuable progress, not make-work refactors.
5. **Green / Yellow / Red policy** — codified table (see below).
6. **Automation window report** — required schema + path under
   `.agent/automation_runs/YYYY-MM-DD_<WINDOW>_REPORT.md`.
7. **Queue + blockers** — `.agent/automation_queue.md` shape, `.agent/blockers.md` shape.
8. **Daily review workflow** — what the human + ChatGPT do at the
   start of Non-Automation Time.
9. **Stop conditions** — exhaustive list (see below).
10. **Non-goals** — explicit list of things AgentOps-2a does NOT cover.

#### Green / Yellow / Red table (must appear verbatim or close)

| Level | Examples | Automation can… |
|---|---|---|
| **Green** | docs, `.agent/*`, daily summaries, diagrams, small local helpers | Run end-to-end without human; merge + push allowed within window |
| **Yellow** | eval sets, taxonomy docs, non-runtime data, small UI tweaks | Execute + write RUN_REPORT/DECISION; merge + push need human GO (defer until Non-Automation Time unless policy explicitly allows) |
| **Red** | classifier.py · extraction.py · schema · prompts · `src/lib/prompts.ts` · sources.yaml · `.github/workflows/**` · generated corpus · production bundle · model selection · deployment config · auth/payment/DB · OpenAI API introduction · autonomous daemon / scheduler | STOP. Write to `blockers.md`. Wait for human approval. |

#### Stop conditions (must appear verbatim or close)

Automation MUST stop when any of:

- A red-zone file would need to change.
- Repo state is dirty unexpectedly (untracked / uncommitted before run start).
- Validation fails.
- Branch cannot fast-forward.
- Scope expands beyond the TASK's allowed/forbidden lists.
- Generated corpus would change unexpectedly.
- Prompt / schema / classifier / extractor / source / cron would change.
- OpenAI API would be needed.
- Deployment would be triggered manually.
- Codex or Claude Code expresses low confidence ("I'm unsure if X").
- Task lacks a clear TASK file (no `.agent/tasks/...` to anchor scope).

### File 2 · `.agent/templates/automation_window_report_template.md` (~120-180 lines)

Schema for the per-window report. Fields the report MUST include:

- `window_name` (`workday_2026-06-30_09-19` / `sleep_2026-06-30_01-09` / `weekend_2026-06-30`)

- `files_changed` (per repo)

- `failed_validations` (count + summary)
- `merge_push_deploy_status` (per repo)
- `recommended_human_decisions`
- `next_suggested_automation_tasks`
