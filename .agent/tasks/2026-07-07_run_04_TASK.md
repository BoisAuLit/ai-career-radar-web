## Objective

Prevent **empty or near-empty PDF
extraction results** from silently
populating the résumé field and
producing low-quality generated
reports. Add a **client-side
minimum-character threshold** on the
extracted-text length. Above the
threshold: keep the existing
successful-populate behavior. Below
the threshold: **do NOT overwrite
the résumé state**, show a
conservative amber warning telling
the user the PDF may be scanned or
image-only, and ask them to paste
text manually or upload a text-
selectable PDF. No backend
extraction changes; no OCR; no new
dependency.

- Threshold: **`MIN_EXTRACTED_RESUME_CHARS = 300`**.
  Justification: 300 characters is
  well below the length of a
  meaningful résumé (a typical
  résumé's plain-text extraction
  is thousands of characters) but
  is comfortably above the noise
  from a scanned-only PDF (which
  typically yields <50 chars of
  garbage from the header
  metadata). If real usage shows
  false positives (e.g., a
  legitimately tiny résumé that
  users think is fine), a future
  yellow loop can tune the value.
- **Do NOT** silently overwrite
  the existing `resume` state
  when below threshold. If
  `resume.trim().length > 0`
  before upload, that content
  MUST remain visible.
- **Do NOT** call the below-
  threshold extraction a
  "successful" upload. In
  particular, do NOT set
  `pdfFilename` in a way that
  looks like the résumé is now
  loaded; the warning must be
  the dominant signal.
- **Do NOT** interfere with
  legitimate manual paste. The
  résumé `<textarea>` `onChange`
  must continue to accept
  arbitrary user input, and the
  warning must clear when the
  résumé state's trimmed length
  crosses back above the
  threshold.
- **Do NOT** block the Generate
  button beyond the existing
  `!resume.trim()` guard —
  disabling the button on an
  empty résumé is already the
  behavior; the new warning is
  the fix-nudge.
- Exact suggested warning copy:
  `"This PDF produced very little
  readable text. It may be scanned
  or image-only. Please paste
  your resume text manually or
  upload a text-selectable PDF
  before generating a report."`
