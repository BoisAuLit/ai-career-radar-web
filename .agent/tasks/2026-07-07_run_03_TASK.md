## Objective

Add a **server-emitted stream-complete
sentinel** and a **client-side conservative
incomplete-report warning banner** so
partial/truncated streamed reports are never
silently treated as complete. The sentinel
is application-level (not model-generated),
is stripped before rendering, and is
never included in prompts. If the browser
reader closes without ever seeing the
sentinel, a modest amber banner appears
next to the report card telling the user
the report may be incomplete and offering
retry.

- P2.1a audit memo

  §8 named silent stream truncation as the

  "Candidate 1 — stream-complete sentinel +

  first fix.
- Current server (`src/app/api/generate-report/route.ts:94-100`)
  returns `streamText().toTextStreamResponse()`
  as a raw text stream. Client

  with no user-visible signal.
