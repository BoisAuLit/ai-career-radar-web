## Objective

Add two small parallel helpers — `.agent/scripts/new_run_report.py` and
`.agent/scripts/new_decision.py` — that scaffold the next RUN_REPORT and
DECISION files from their respective templates. Same shape as
`new_task.py`: standard-library-only, local-only, refuse-to-overwrite,
single-line stdout, never auto-commit, never invoke external commands.

The goal is to remove the same manual ritual at the *end* of each
AgentOps run that `new_task.py` already removed at the start. **No
orchestration.** These are file-template helpers, not a loop driver.

## Background

AgentOps-1a → 1b proved the loop:

```
TASK (new_task.py)  →  impl  →  RUN_REPORT (manual)  →  DECISION (manual)
```

The two trailing manual steps still cost ~3 minutes each (copy template,
substitute the metadata block, find the right `based_on_run_report`
path). This task adds the symmetrical helpers so a full loop becomes:

```
TASK (new_task.py)  →  impl  →  RUN_REPORT (new_run_report.py)  →  DECISION (new_decision.py)
```

These helpers do **not** write the actual report/decision content — they
only create correctly-named files with the metadata block resolved and
the rest of the template left as placeholders for Claude / ChatGPT /
human to fill in.

This task is intentionally scoped to the two helpers. **It is NOT**:
- a general orchestrator
- a watcher / daemon / dashboard
- anything that calls Claude, OpenAI API, or any external service
- anything that touches GitHub Actions

If during execution the design starts drifting toward any of the above,
stop and surface in the RUN_REPORT instead of continuing.

## Constraints

- **Python standard library only.** No `pip install`, no `requirements.txt`,
  no third-party imports. Acceptable: `argparse`, `pathlib`, `re`,
  `datetime`, `sys`, `os`. Anything else stop and surface.
- **No new dependencies.** `package.json` / `package-lock.json`
  byte-identical to `HEAD` at the end of the run.
- **No npm script wrapper.**
- **No app code touched.**
- **No prompt / classifier / extractor / model / cron / pipeline edits.**
- **No OpenAI API integration.** No code that imports an OpenAI SDK, no
  HTTP requests to OpenAI endpoints, no `.env` keys, no client
  initialization. AgentOps stays manual / file-driven until a separate
  phase (AgentOps-2+) explicitly opens that door with its own design memo.
- **No GitHub Actions or `@claude` integration.**
- **Local-only.** Neither script may execute any external command:
  - no `subprocess` / `os.system` / `os.popen` / shell invocation
  - no HTTP requests / network calls of any kind
  - no `git push` / `git commit` / `git add`
  - no Claude / OpenAI / model-API call
  - no `npm`, `vercel`, `gh`, `deno`, anything
- **No autonomous loop, daemon, file watcher, dashboard, scheduler.**
  Each script runs once, produces one file, exits.
- **Refuse to overwrite.** If the target file already exists, exit
  non-zero with a clear stderr message. Never clobber.
- **Do not push.** Generated RUN_REPORT / DECISION files are not
  auto-committed by the script. Human / Claude commits them explicitly.
- **Editor placeholders preserved.** Output must still look like the
  template — `<placeholder>` markers left intact so the author (Claude
  for RUN_REPORT, ChatGPT / human for DECISION) can fill them in. Only
  the metadata block at the top is resolved.

## Implementation outline (for the executor)

Both scripts mirror the shape of `new_task.py`. Pseudo-code:

```python
# new_run_report.py
"""Scaffold the next .agent/run_reports/YYYY-MM-DD_run_XX_RUN_REPORT.md
from .agent/templates/run_report_template.md.

Local-only, stdlib-only, no side effects beyond writing one file.
"""

# args:
#   --task-id YYYY-MM-DD_run_XX        OR
#   --date YYYY-MM-DD + --run NN       (both required if --task-id absent)

# steps:
#   1. parse + validate args (date format, run 1..99)
#   2. compose task_id and target path .agent/run_reports/{task_id}_RUN_REPORT.md
#   3. if target.exists(): print error, return 2
#   4. read template, substitute metadata block (task_id, date, run_number)
#   5. write target, print absolute path, return 0
```

```python
# new_decision.py
"""Scaffold the next .agent/decisions/YYYY-MM-DD_run_XX_DECISION.md
from .agent/templates/decision_template.md.

Local-only, stdlib-only, no side effects beyond writing one file.
"""

# args:
#   --task-id YYYY-MM-DD_run_XX        OR
#   --date YYYY-MM-DD + --run NN
#   (no extra flags. RUN_REPORT path is auto-derived from task_id.)

# steps:
#   1. parse + validate args
#   2. compose task_id
#   3. target = .agent/decisions/{task_id}_DECISION.md
#   4. if target.exists(): print error, return 2
#   5. read template, substitute:
#        - decision_id  → "{task_id}_DECISION"
#        - based_on_run_report  → ".agent/run_reports/{task_id}_RUN_REPORT.md"
#   6. write target, print absolute path, return 0
```

If both scripts end up sharing >20 lines of identical helpers
(`require_task_id_or_date_run`, `substitute_metadata`, etc.), factor
those into `.agent/scripts/_common.py` (single underscore prefix to
signal "internal, not invoked directly"). Otherwise keep both
self-contained.

Total combined size across the two scripts (excluding shared helper):
**target ≤ 200 lines**. Each script alone should not exceed ~120 lines.
