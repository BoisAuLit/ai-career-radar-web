## Diagnostic design

### A · Network events collection

- Register `page.on("requestfailed", …)` before navigation:
  capture `elapsed_ms · url · method · resource_type · failure_reason`.
- Register `page.on("response", …)` before navigation: for status
  ≥ 400 capture `elapsed_ms · url · status · status_text · method ·
  resource_type · selected safe headers only · sanitized body
  excerpt (≤ 500 chars, HTML stripped)`.
- **Safe-headers allowlist** (do NOT collect anything else):
  `request-id`, `x-request-id`, `anthropic-request-id`, `cf-ray`,
  `traceparent`, `x-vercel-id`, `x-amzn-trace-id`, `retry-after`.
- **Explicitly NEVER**: `authorization`, `cookie`, `set-cookie`,
  API keys, request payloads, resume text, cookies.

### B · Fail-fast completion race

- Replace the current success-only `waitForSelector('button:has-text("Copy report")', …)`
  with a Promise.race between:
  - **success**: `button:has-text("Copy report")`
  - **error**: `button:has-text("Retry")` (stable inside
    `stage === "error"` panel)
- Both waits use the **existing** `HARD_LATENCY_MS`.
- If success wins → `completion_state = "success"` · `doneReached = true`.
- If error wins → `completion_state = "application_error"` ·
  `doneReached = false` · capture short visible error text
  (sanitized · ≤ 500 chars).
- If neither wins in 240 s → `completion_state = "hard_timeout"` ·
  `doneReached = false`.
