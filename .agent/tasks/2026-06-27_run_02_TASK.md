## Objective

Add a small local helper that reduces manual work when starting a new
AgentOps task. The helper generates the next
`.agent/tasks/YYYY-MM-DD_run_XX_TASK.md` file from
`.agent/templates/task_template.md`, using today's date (or `--date`) and the
next available run number for that date. Goal: cut ~3 minutes off the start
of every future TASK and remove the off-by-one risk on the run-number
sequence.

## Background

AgentOps-1a proved the file-based loop works end-to-end:
TASK → implementation → RUN_REPORT → DECISION → human approval → merge →
push → production verification (see commits `8c4422e`, `11ecf52`, `77a8e70`,
`1689a10`, `265e285`, `4268109`).

The remaining friction is the manual ritual at the start of each TASK:
copy the template, replace the metadata block, find the next free run
number for today, save under the right path. That is exactly the kind of
mechanical setup work a small script removes without introducing any
automation surface.

This is **AgentOps-1b**. It must stay local-only and standard-library-only
per the AgentOps non-goals in `.agent/README.md`. Anything that talks to
the network, an LLM, GitHub Actions, a daemon, or a UI wrapper is out of
scope and belongs in a later AgentOps-N phase with its own design memo.

## Constraints

- **Standard library only.** No `pip install`, no `requirements.txt`, no
  third-party imports. Acceptable: `argparse`, `pathlib`, `re`,
  `datetime`, `sys`, `os`. Anything else stop and surface.
- **No new dependencies.** `package.json` / `package-lock.json`
  byte-identical to `HEAD` at the end of the run.
- **No npm script wrapper.** Do not add `"new:task"` to
  `package.json`; invocation is `python .agent/scripts/new_task.py` only.
- **No app code touched.**
- **No prompt / classifier / extractor / model / cron / pipeline edits.**
- **Local-only.** The script must not execute any external command:
  - no `subprocess` calls to `git`, `gh`, `vercel`, `npm`, `node`, etc.
  - no HTTP requests
  - no `os.system` or shell invocation
  - no `git push` / `git commit` / `git add`
  - no Claude or OpenAI API call
- **No autonomous loop / daemon / watcher.** The script runs once,
  produces one file, exits.
- **No GitHub Actions.** The script is not invoked by CI.
- **Refuse to overwrite.** If the target file already exists, the script
  must exit non-zero with a clear message; never clobber.
- **Do not push.** The created TASK file should NOT be auto-committed by
  the script. The human (or follow-up Claude TASK) commits it explicitly.
- **Editor placeholders preserved.** The output should still look like
  the template — `<one-line title>`, `<criterion 1>` etc. left as
  placeholders so the author can fill them in, with only the metadata
  block at the top resolved (task_id, date, run_number, title).

## Implementation outline (for the executor)

Create `.agent/scripts/new_task.py` with roughly this shape (the
executor may tighten as needed):

```python
#!/usr/bin/env python3
"""Generate the next .agent/tasks/YYYY-MM-DD_run_XX_TASK.md from the template.

Local-only, standard-library-only, no side effects beyond writing one file.
"""
import argparse, datetime, pathlib, re, sys

REPO_ROOT = pathlib.Path(__file__).resolve().parents[2]   # .agent/scripts/ → repo root
TEMPLATE = REPO_ROOT / ".agent" / "templates" / "task_template.md"
TASKS_DIR = REPO_ROOT / ".agent" / "tasks"

def next_run_number(date_str: str) -> int:
    pattern = re.compile(rf"^{re.escape(date_str)}_run_(\d{{2}})_TASK\.md$")
    existing = []
    if TASKS_DIR.is_dir():
        for p in TASKS_DIR.iterdir():
            m = pattern.match(p.name)
            if m:
                existing.append(int(m.group(1)))
    return (max(existing) + 1) if existing else 1

def render(template_text: str, *, task_id, date, run_number, title, risk) -> str:
    # Replace the four metadata lines + risk_level line, leave the rest as-is.
    out = template_text
    out = re.sub(r"- \*\*task_id\*\*: `[^`]+`", f"- **task_id**: `{task_id}`", out, count=1)
    out = re.sub(r"- \*\*date\*\*: `[^`]+`", f"- **date**: `{date}`", out, count=1)
    out = re.sub(r"- \*\*run_number\*\*: `[^`]+`", f"- **run_number**: `{run_number:02d}` (zero-padded sequence within the day)", out, count=1)
    if title:
        out = re.sub(r"- \*\*title\*\*: <[^>]+>", f"- **title**: {title}", out, count=1)
    if risk:
        out = re.sub(r"`green` \| `yellow` \| `red`", f"`{risk}`", out, count=1)
    return out

def main(argv=None):
    p = argparse.ArgumentParser(description="Scaffold the next AgentOps TASK file.")
    p.add_argument("--date", help="YYYY-MM-DD (default: today, local)")
    p.add_argument("--title", help="Optional short task title to seed the file")
    p.add_argument("--risk", choices=("green", "yellow", "red"), help="Optional risk level")
    args = p.parse_args(argv)

    date_str = args.date or datetime.date.today().strftime("%Y-%m-%d")
    try:
        datetime.date.fromisoformat(date_str)
    except ValueError:
        print(f"error: --date must be YYYY-MM-DD, got {date_str!r}", file=sys.stderr)
        return 2

    n = next_run_number(date_str)
    task_id = f"{date_str}_run_{n:02d}"
    target = TASKS_DIR / f"{task_id}_TASK.md"
    if target.exists():
        print(f"error: refuse to overwrite existing {target}", file=sys.stderr)
        return 2

    template_text = TEMPLATE.read_text()
    rendered = render(template_text, task_id=task_id, date=date_str, run_number=n,
                      title=args.title, risk=args.risk)
    target.parent.mkdir(parents=True, exist_ok=True)
    target.write_text(rendered)
    print(target)
    return 0

if __name__ == "__main__":
    raise SystemExit(main())
```

The executor may adapt the regex strategy (e.g., a more robust placeholder
system) but the contract is: same inputs → same output, idempotent failure
mode on collision, no side effects beyond writing the one TASK file.
