## R2 tier — eight strict conditions

The new `terminal_punctuation_only` match tier applies **only when
all of these hold**:

1. **Source id resolves** in corpus.
2. **Company check passes** (case+punctuation-insensitive
   containment).
3. **Role/title check is `pass` or `unknown`, not `fail`**.
4. **The only unmatched difference is the final punctuation mark**
   (verified by matching `quote[:-1]` verbatim/normalized/CI in the
   source body).
5. **The rest of the quote matches verbatim, normalized, or
   case-insensitive normalized**.
6. **No word has been inserted, deleted, reordered, or replaced**
   (guaranteed by substring match of `quote[:-1]`).
7. **Quote length ≥ 40 non-space characters** (threshold from 5c
   DECISION recommendation).
8. **Punctuation change does not change meaning**:
   - **Allowed**: source `,` / `;` / no-final-punct → report `.`
   - **Forbidden**: source `?` / `!` / `:` → report `.`
   - Only report-side terminal `.` is considered (a report ending in
     `?` / `!` / other is not eligible for this micro-tier).

If any condition fails, the tier does NOT fire and the previous
classification stands.

## Priority order in checker

For each Evidence quote with cited `jd_id`:

1. **Verbatim** substring → GREEN
2. **Normalized** substring → AMBER `normalized`
3. **Case-insensitive normalized** substring → AMBER `case_insensitive`
4. **Terminal-punctuation-only** (new R2 tier) → AMBER
   `terminal_punctuation_only`
5. Quote contains `...` / `…`: ellipsis matcher → AMBER
   `ellipsis_fragments` (all fragments match) OR RED
   `unmatched_ellipsis_fragment` (any fragment unmatched)
6. Else → RED `unmatched`

R2 is placed before the ellipsis matcher because the ellipsis
matcher fallback still triggers only when quote has `...`/`…`; for a
non-ellipsis quote with only terminal-punct swap, R2 will fire and
the ellipsis matcher is skipped.
