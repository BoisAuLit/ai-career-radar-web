## Parser algorithm (per 5a design memo §7)

1. Load report text.
2. Extract **Evidence Appendix** table (tab-separated
   `JD_ID\tCOMPANY\tTITLE` under the `Evidence Appendix` header).
3. Extract **quote candidates**:
   - explicit `Evidence quote: "..." — <Company>, <jd_id>.` (primary
     evidence pattern)
   - straight double-quoted spans elsewhere in report (informational,
     tracked but not blocking)
4. For each **evidence quote**, capture:
   - snippet first 60 chars (whitespace normalized)
   - char length
   - nearby cited company + `jd_id`
5. **Normalize** text: collapse whitespace · normalize curly → straight
   quotes · normalize dashes minimally · preserve word order.
6. **Match** each evidence quote against the cited record's `body`:
   - verbatim (in normalized-newlines body) → GREEN mark
   - normalized only → AMBER mark
   - no match after normalization → RED mark
   - if `jd_id` does not resolve in corpus → RED mark
7. **Metadata checks**:
   - cited company vs corpus company (case-insensitive) → RED if mismatch
   - cited title/role vs corpus title (loose containment) → AMBER if
     mismatch, RED if clearly incompatible archetype
8. **Evidence Appendix checks**:
   - if appendix absent and evidence claimed anywhere in report → RED
   - if appendix present but no extractable evidence quotes in body →
     AMBER
9. **Duplicate checks**: count identical evidence quote snippets.
10. **Emit `qi_summary.json`** per 5a design memo §7 schema envelope.

## Verdict rollup

- Any RED → `red`
- Otherwise any AMBER → `amber`
- Otherwise → `green`
- If report unavailable → `blocked_no_report`
